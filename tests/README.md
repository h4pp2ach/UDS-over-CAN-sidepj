# 테스트 주석 규칙

테스트는 기능 단위로 묶고, 각 묶음 바로 위에 다음 두 줄의 한국어 주석을 유지한다.

- `# 기능 그룹:`에는 테스트 대상이 공유하는 책임이나 기능명을 적는다.
- `# 검증 목적:`에는 해당 묶음이 보장하는 동작과 실패 조건을 한 문장으로 적는다.
- 새 테스트는 기존 기능 그룹에 배치한다. 적절한 그룹이 없다면 같은 형식의 새 그룹을 만든다.
- 구현 절차보다 외부에서 관찰할 수 있는 동작을 중심으로 작성한다.

예시:

```python
# 기능 그룹: ISO-TP Flow Control 처리
# 검증 목적: CTS, WAIT, OVERFLOW 상태에 따라 송신 흐름이 올바르게 진행되거나 중단되는지 확인한다.
def test_sender_continues_after_cts():
    ...


def test_sender_stops_after_overflow():
    ...
```
